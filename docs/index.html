üìã Core Concepts

1. Data Context

Defines storage location for metadata, configurations, and validation results

üí° All GX scripts start with creating a Data Context
context = gx.get_context()

2. Data Sources

Tell GX where your data is located and how to connect to it

Amazon S3 Azure Blob Google Cloud Local Files 
source_folder = "data/"
data_source_name = "resale_flat"

data_source = context.data_sources.add_pandas_filesystem(
    name=data_source_name, 
    base_directory=source_folder
)

3. Data Assets

A collection of related records within a Data Source

asset_name = "resale_csv_files"
file_csv_asset = data_source.add_csv_asset(name=asset_name)

4. Batch Definition

Request all records or a subset from a Data Asset

batch_definition = file_data_asset.add_batch_definition_path(
    name="resale_flat_201701_202306.csv",
    path="resale_flat_201701_202306.csv"
)

batch = batch_definition.get_batch()

‚úÖ Creating Expectations

Single Expectation

preset_expectation = gx.expectations.ExpectColumnMaxToBeBetween(
    column="lease_commence_date",
    min_value=1,
    max_value=2020
)

validation_results = batch.validate(preset_expectation)

Expectation Suite

üí° Group multiple expectations together for comprehensive validation
suite_name = "sctp_expectation_suite"
suite = gx.ExpectationSuite(name=suite_name)
suite = context.suites.add(suite)
suite.add_expectation(preset_expectation)

Validation Definition

definition_name = "sctp_validation_definition"
validation_definition = gx.ValidationDefinition(
    data=batch_definition,
    suite=suite,
    name=definition_name
)

validation_results = validation_definition.run()

‚úÖ dbt_utils Tests

accepted_range Test

Check if values are within an acceptable range

- name: date
  tests:
    - dbt_utils.accepted_range:
        min_value: "PARSE_DATE('%F', '2012-01-01')"
        max_value: "CURRENT_DATE()"
üí° Great for validating date ranges or numeric boundaries

expression_is_true Test

Validate complex business logic across multiple columns

tests:
  - dbt_utils.expression_is_true:
      expression: "ROUND(sale_dollars, 1) = TRUNC(bottles_sold * state_bottle_retail, 1)"
üí° This is a model-level test - place it outside the columns section

üöÄ Why Orchestration?

Orchestration allows you to automate your data pipelines, schedule testing workflows, manage dependencies, and monitor data quality continuously in production environments.

üì¶ Dagster for Data Testing

üí° Dagster provides powerful capabilities for orchestrating both Meltano and DBT pipelines with built-in testing support

Sample Dagster Configuration

from dagster import Definitions, ScheduleDefinition, job
from dagster_meltano import meltano_resource, meltano_run_op

@job(resource_defs={"meltano": meltano_resource})
def run_elt_job():
    tap_done = meltano_run_op("tap-github target-bigquery")()

# Schedule to run daily at midnight
elt_schedule = ScheduleDefinition(
    job=run_elt_job,
    cron_schedule="0 0 * * *",
)

defs = Definitions(
    jobs=[run_elt_job],
    schedules=[elt_schedule],
)

üìä Dagster Configuration Example

from dagster import Definitions, ScheduleDefinition, job
from dagster_dbt import dbt_cli_resource, dbt_run_op, dbt_test_op

@job(resource_defs={"dbt": dbt_cli_resource})
def dbt_pipeline():
    dbt_run = dbt_run_op()
    dbt_test_op(start_after=dbt_run)

# Schedule to run daily
dbt_schedule = ScheduleDefinition(
    job=dbt_pipeline,
    cron_schedule="0 2 * * *",  # 2 AM daily
)

defs = Definitions(
    jobs=[dbt_pipeline],
    schedules=[dbt_schedule],
)

This sets up a complete DBT pipeline with automated testing that runs daily at 2 AM
